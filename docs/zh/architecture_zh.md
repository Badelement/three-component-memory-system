# 架构设计

## 系统概述

三组件记忆系统是一个为 OpenClaw 设计的模块化、可扩展的记忆管理解决方案。它结合了三个专门的存储系统，提供全面的记忆功能。

## 组件详情

### 1. LanceDB 组件（语义层）

#### 目的
- 基于语义含义存储和搜索记忆
- 启用"查找相似"功能
- 理解概念之间的上下文和关系

#### 实现
- **数据库**: LanceDB（本地向量数据库）
- **嵌入模型**: all-MiniLM-L6-v2（384 维）
- **存储格式**: 记忆内容的向量嵌入
- **搜索算法**: 余弦相似度与近似最近邻

#### 性能特征
- **索引时间**: 每个记忆约 10ms
- **搜索时间**: 每个查询 5-15ms
- **存储**: 每个记忆约 1.5KB（384 维 float32 向量）

### 2. SQLite 组件（结构化层）

#### 目的
- 存储结构化记忆元数据
- 支持快速属性查询
- 提供事务完整性和数据持久性

#### 实现
- **数据库**: SQLite 3
- **表结构**: 
  - `memories`（记忆内容、分类、重要性、时间戳）
  - `tags`（标签系统）
  - `categories`（分类管理）
- **索引**: 在常用查询字段上建立索引

#### 性能特征
- **查询时间**: 简单查询 <1ms，复杂查询 <10ms
- **存储**: 每个记忆约 2-5KB（取决于内容长度）
- **并发**: 支持多线程读取，单线程写入

### 3. NetworkX 组件（关系层）

#### 目的
- 发现和存储记忆之间的关系
- 支持基于关系的查询
- 提供记忆网络的洞察

#### 实现
- **图库**: NetworkX
- **节点**: 记忆条目
- **边**: 基于相似度、时间接近性、共享标签的关系
- **算法**: 社区检测、最短路径、中心性分析

#### 性能特征
- **图构建**: 每个关系约 5ms
- **查询时间**: 简单图查询 10-50ms
- **内存使用**: 每个节点约 500 字节，每条边约 200 字节

## 数据流

### 1. 记忆添加流程
```
用户输入 → 重要性评估 → 向量嵌入生成 → 三组件存储
       ↓           ↓              ↓
   重要性过滤  文本处理      并行存储到：
                       1. LanceDB（向量）
                       2. SQLite（结构化）
                       3. NetworkX（关系）
```

### 2. 记忆搜索流程
```
用户查询 → 查询解析 → 并行搜索 → 结果融合 → 排序返回
                  ↓         ↓         ↓
               语义搜索  文本搜索  关系搜索
               (LanceDB) (SQLite) (NetworkX)
```

### 3. 自动记录流程
```
对话消息 → 重要性评分 → 阈值检查 → 自动存储
      ↓           ↓           ↓
  内容分析  关键词检测  配置检查
```

## 集成设计

### OpenClaw 集成
- **技能接口**: 标准的 OpenClaw skill 结构
- **命令系统**: `/memory` 命令前缀
- **配置管理**: 通过 OpenClaw config.json 配置
- **事件处理**: 响应 OpenClaw 系统事件

### API 设计原则
1. **一致性**: 统一的 MemorySystem 类接口
2. **可扩展性**: 插件式组件设计
3. **性能**: 异步操作和缓存优化
4. **容错**: 优雅降级和错误恢复

## 性能优化

### 缓存策略
- **向量缓存**: 最近使用的嵌入向量
- **查询缓存**: 常用查询结果
- **图缓存**: 频繁访问的子图

### 索引优化
- **SQLite**: B-tree 索引在常用查询字段
- **LanceDB**: IVF-PQ 索引加速向量搜索
- **NetworkX**: 预计算的图度量

### 批量操作
- **批量添加**: 减少数据库事务开销
- **批量搜索**: 并行化多个查询
- **批量更新**: 原子性操作组

## 扩展性设计

### 水平扩展
- **分片策略**: 按用户或时间分片
- **复制策略**: 主从复制配置
- **负载均衡**: 查询分发机制

### 垂直扩展
- **组件替换**: 可替换单个存储组件
- **插件系统**: 自定义处理管道
- **模型切换**: 支持不同嵌入模型

## 监控与维护

### 健康检查
- **组件状态**: 定期检查各组件可用性
- **性能指标**: 响应时间、内存使用、存储空间
- **数据完整性**: 一致性验证和修复

### 维护任务
- **数据清理**: 过期记忆归档
- **索引重建**: 定期优化索引
- **备份恢复**: 自动备份和恢复机制

---

**架构设计目标**: 在性能、可扩展性和易用性之间取得平衡，为 OpenClaw 提供强大的长期记忆能力。
